# Shield Memory Log

> **Purpose:** Every verification action, missed bug, or false positive is logged here. Before running any validation, the Shield MUST read this file and ensure past mistakes are not repeated.

## Format

Each entry follows this structure:

```
### [YYYY-MM-DD] <short title>
- **Action:** What was tested / verified
- **Consequence:** What happened as a result (positive or negative)
- **Lesson:** What to do (or avoid) next time
- **Severity:** low | medium | high | critical
```
